# Data Model: Docusaurus Frontend with Better Auth Authentication

## User Entity
- **id**: string (primary key, auto-generated by Better Auth)
- **email**: string (required, unique, valid email format)
- **password**: string (required, hashed by Better Auth)
- **name**: string (required, user's display name)
- **hardwareBackground**: string (optional, user's hardware experience)
- **softwareBackground**: string (optional, user's software experience)
- **createdAt**: Date (auto-generated by Better Auth)
- **updatedAt**: Date (auto-generated by Better Auth)
- **emailVerified**: boolean (auto-managed by Better Auth)

## Session Entity (Managed by Better Auth)
- **sessionId**: string (unique identifier)
- **userId**: string (foreign key to User)
- **expiresAt**: Date (session expiration)
- **createdAt**: Date (session creation time)
- **ipAddress**: string (optional, for security tracking)
- **userAgent**: string (optional, for security tracking)

## Authentication State (Frontend Context)
- **user**: User object or null (current authenticated user)
- **isLoading**: boolean (whether auth state is being determined)
- **isAuthenticated**: boolean (whether user is currently authenticated)
- **error**: string or null (any authentication errors)

## Validation Rules
- Email must be in valid format (user@domain.tld)
- Password must meet strength requirements (min 8 characters, 1 uppercase, 1 lowercase, 1 number)
- Name must be 2-50 characters
- Hardware/Software background fields are optional, max 500 characters each

## State Transitions
1. **Unauthenticated → Authenticating**: User submits login form
2. **Authenticating → Authenticated**: Login successful
3. **Authenticating → Unauthenticated**: Login failed
4. **Authenticated → Unauthenticated**: User signs out or session expires
